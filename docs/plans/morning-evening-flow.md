# 毎日の自動フロー：朝の進捗確認 → PMレポート → 夜の振り返り

このドキュメントでは、Slackボットが毎日自動で行う3つの業務フローを説明します。

---

## 1. 朝の進捗確認（毎朝 9:00）

### やること

毎朝9時に、ボットが各メンバーにSlack DMで「今日のタスク一覧」を送ります。

### 流れ

1. ボットがNotionから今のスプリントのタスクを自動で取得します
2. メンバーごとに担当タスクを整理し、AIが以下の情報をまとめます
   - スプリント全体の進み具合（計画・完了・残りのポイント）
   - 状況の判定（🟢順調 / 🟡注意 / 🔴危険）
   - 注意すべきタスク（期限切れ、止まっているものなど）
3. 各メンバーにDMが届きます。内容は以下のような構成です
   - 挨拶 ＋ 今日のポイントまとめ
   - 【期限超過】【今週期限】【進行中・その他】に分けたタスク一覧
4. 9:10〜9:50の間、返信がないメンバーには10分ごとにリマインドが届きます

### メンバーがやること

- DMに返信すれば、リマインドは止まります（返信内容は後のPMレポートに使われます）

### うまくいかなかった場合

- 補助データ（Googleスプレッドシートなど）が取れなくても、Notionのタスクだけでレポートは作られます
- Notionのデータ自体が取れなかった場合は、PMOチャンネルにエラー通知が届きます

---

## 2. PMレポートの生成・送信（毎朝 10:00）

### やること

朝のDMで集まったメンバーの返信と、最新のタスクデータをもとに、PM向けのデイリーレポートを自動生成して送ります。

### 流れ

1. ボットが各メンバーの返信内容をまとめます
2. Notionからタスクを再取得し、前日からの変化を確認します
   - 昨日消化したポイント
   - 2日以上動きのないタスク
3. AIがこれらをまとめて、PMレポートを作成します
   - 【スケジュール分析】：計画 vs 実績、このペースで間に合うかの判定
   - 【メンバー稼働余力】：各メンバーの残タスク量と完了見込み
   - 【今週期限のタスク】：直近3日以内に期限が来るタスク一覧
4. PMOチャンネルにレポートが投稿されます
5. PMがレポートに返信しない場合、11:00〜19:00の間、1時間ごとにリマインドが届きます

### うまくいかなかった場合

- メンバーの返信が1件もなくても、レポートは作成されます（「返信なし」と記載されます）
- Notionのデータ取得に失敗した場合は、PMOチャンネルにエラー通知が届きます

---

## 3. 夜のステータス更新確認（夕方）

### やること

夕方に、各メンバーへ「今日の作業はNotionに反映しましたか？」と確認のDMを送ります。

### 流れ

1. ボットが各メンバーのDMに確認メッセージを送ります
2. メッセージには3つのボタンが付いています
   - 「✅ 更新済み」：今日の作業はもうNotionに反映した
   - 「🔄 作業中」：まだ作業中なので、あとで更新する
   - 「🚫 今日は進捗なし」：今日は特に進捗なし
3. ボタンを押すと、回答内容に応じてNotionのタスクステータスが自動更新されます

### うまくいかなかった場合

- ボタンが押されなかった場合は、翌朝のフローで「未更新タスク」として検出されます
